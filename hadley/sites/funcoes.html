Funções

Introdução

Até este momento você já viu variáveis, expressões e if else e criou alguns programas. Entretanto, para poder escrever programas mais complexos, você ainda precisa aprender os seguintes elementos:

  • Funções
  • Laços de repetição
  • Vetores
  • Objetos

Neste tutorial nós iremos aprender funções: o que são, para que servem e como utilizar.

O que são funções?

Você deve lembrar de ter visto algo assim em matemática:

$$f(x) = x + 2$$

Isso é uma função. Uma função é dividida nas seguintes partes:

  • Nome da função
  • Parâmetros
  • Corpo

No exemplo acima, o nome da função é f. Os parâmetros são todas as variáveis dentro dos parenteses, no caso só existe um parâmetro que é a variável x e o corpo da função é a expressão $x + 2$

Agora olhe o seguinte exemplo:

$$calcular(nota, faltas) = nota \times 100 - faltas \times 0.1$$

Nesse exemplo o nome da função é calcular e ela possui dois parâmetros: nota e faltas que são as variáveis que estão dentro dos parenteses. Por fim, o corpo da função é a expressão $nota \times 100 - faltas \times 0.1$.

Hey, mas aqui a gente está programando e não sendo um matemático, correto? Então vamos olhar como é uma função em programação. Em javascript uma função é assim:


function nome(parametro1, parametro2, ..., parametroN) {
    corpo
}

Em javascript usamos a palavra especial function para indicar que estamos criando uma função. Em seguida vem o nome da função, depois seus parâmetros e, por fim, o corpo da função que deve estar entre as chaves { }.

Se fossemos reescrever os exemplos de funções matemáticas em javascript, ficaria assim:


function f(x) {
    return x + 2;
}

function calcular(nota, faltas) {
    return nota * 100 - faltas * 0.1;
}

Bem, apareceu uma nova palavra: return. Esse return serve para... retornar um valor! Que legal, mas você não sabe ainda ao certo o porquê de retornar um valor, mas vai saber lá na frente. Só queria te mostrar como são as funções em javascript. Então lembra disso: funções são compostas por: nome, parâmetros e corpo. ok?

Agora vamos esquecer a parte matemática por um instante porque as funções em programação vão começar a ser bem diferentes do que são em matemática. Sabe tudo o que você aprendeu até agora (variáveis, expressões, if else)? Tudo o que você aprendeu até agora pode ser usado como corpo de uma função em programação. Sabe todos aqueles exercícios que você resolveu até agora? Cada um deles poderia ser colocado em uma função. A pergunta é: por que raios eu iria querer colocar dentro de uma função?

Para que servem funções

Você pode não saber ainda, mas funções são extremamente importantes e essenciais para escrever programas complexos e objetivo dessa seção é tentar te explicar o porquê.

Primeiro vamos tentar ver a coisa de um ponto de vista prático. Abra aí o vim, escreva qualquer programa que te vier a mente e execute ele. Você vai perceber que sempre, SEMPRE, ele começa executando a partir da primeira instrução e vai descendo até chegar a última.

Na verdade, nem todas as instruções são executadas caso exista if else. Se a condição do if for verdadeira, só o que estiver dentro do if é que será executado. Caso contrário, só o que está dentro do else é que será executado, não é verdade? Mas em ambos os casos, a execução vai vindo de cima para baixo.

E por que estou falando isso? Porque com funções você vai poder mudar essa ordem de execução! Um dos objetivos práticos de uma função é permitir que um trecho de código seja chamável. Esse trecho de código é justamente o corpo da função (hey, você ainda lembra o que é corpo da função, não é?)

E o que significa ser chamável? Significa que o código só vai ser executado se e apenas se você chamar ele.

Você pode não saber, mas já vem usando funções desde o começo e não sabia. Por acaso você já usou alguma das funções abaixo?


nome = prompt('Digite seu nome');
log('Uma mensagem');
alert('Mais uma mensagem');

Pois é, aí foram chamadas (chamável, sacou?) as funções prompt, log e alert, mais conhecidas por vocês como window.prompt, console.log e window.alert.

Funções são feitas para serem chamadas e você as chama quando é necessário. Por exemplo, quando é que você precisa chamar a função prompt? Quando você precisa pedir algum dado pro usuário. E quando é que você chama o log ou alert? Quando você precisa mostrar uma mensagem pro usuário, não é mesmo?

O que você vai aprender daqui em diante é como criar as suas próprias funções.

Por que criar minhas próprias funções?

Existem algumas razões para se criar funções. Em resumo, as principais são:

  • Ajudar a concentração
  • Reutilizar código

O ser humano tem uma capacidade limitada de concentração, não sendo capaz de fazer várias coisas ao mesmo tempo. Nós, humanos, trabalhamos melhor fazendo uma ou poucas tarefas de cada vez. Além disso, nós gostamos de ter um mńimo de organização em nossas vida. Olha só alguns exemplos:

  • Livros são dividos em capítulos, seções, subseções etc
  • Casas são dividas em quartos, cozinha, sala etc
  • Cidades são dividas por bairros
  • Etc, etc etc

Nós humanos precisamos desse tipo de organização, dessa divisão. Isso nos ajuda a pensar e raciocinar melhor sobre as coisas. Em programação, as funções vão cumprir papel semelhante. Ao invés de vermos o código como uma coisa só, nós vamos poder dividir o código em funções de forma a fazer mais sentido pra gente.

Então o primeiro ponto importante em usar funções é esse: vai nos ajudar na concentração, permitindo dividir o código em partes que faça sentido para a gente.

O outro ponto é a reutilização de código. Você deve lembrar do bom e velho cálculo da média aritmética:

Se tu parar pra prestar atenção, é a mesma lógica sempre: pede o nome, depois as notas, depois calcula a média e mostra o resultado. É aí que entra as funções e sua capacidade de reaproveitar código. Lembra que lá no inicio eu ia te explicar aquela palavra nova, a return? Pois bem, a gente vai começar a usar ela pra valer agora. Olha o código abaixo:


function lf(msg) {
    return parseFloat(window.prompt(msg));
}

Eu criei uma função, chamada lf (abreviação de Ler Float), que na prática vai funcionar como um apelido para parseFloat(window.prompt()). Isso significa que agora ao invés de eu fazer assim:


var nome = window.prompt('Qual o nome do primeiro aluno?');
var n1 = parseFloat(window.prompt('Digite a primeira nota');
var n2 = parseFloat(window.prompt('Digite a segunda nota');
var n3 = parseFloat(window.prompt('Digite a terceira nota');

Eu posso fazer assim:


var nome = window.prompt('Qual o nome do primeiro aluno?');
var n1 = lf('Digite a primeira nota');
var n2 = lf('Digite a segunda nota');
var n3 = lf('Digite a terceira nota');

Ha! Agora o meu código está mais compacto, mais legível. Agora eu não tenho mais que digitar bixiga de parseFloat(window.prompt()) quando quiser ler um float. E aí eu poderia criar funções parecidas para ler inteiro, string etc:


function lf(msg) {
    return parseFloat(window.prompt(msg));
}

function li(msg) {
    return parseInt(window.prompt(msg));
}

function ls(msg) {
    return window.prompt(msg);
}

Bom, agora acho que seria uma boa hora de explicar essa palavra especial return. Não se preocupe que eu acredito que será fácil de entender quando usar ou não ela. Basicamente, uma função pode ou não retornar um valor. Quando você quiser que a função retorne um valor, você usa o return. E quando não quiser que ela não retorne valor... Bom, você já deve imaginar que não é para usar return.

"Ok, mas como é que eu sei que minha função deve retornar um valor?", você deve estar se perguntando. E aí, meus caros, eu acho que achei uma forma prática e sem enrolação de te ensinar isso. A regra é a seguinte:

  • Você vai usar sua função como parte de uma expressão ou num if else? Então deve retornar
  • Você vai chamar a função sozinha? Então não é obrigatório ter return

E para ficar mais claro, vamos para exemplos práticos. Eu vou usar funções ficticias chamadas de fun1, fun2, etc e vou indicar se deve ou não ter retorno. O código do exemplo não tem lógica nenhuma, é só pra gente ver os casos onde se deve ou não usar return.


var v1 = fun1();        // tem return porque eh usado em expressao
var v2 = fun2() + 2;    // tem return porque tb esta em expressao

if (fun3()) {           // deve ter return pq esta sendo usado em if else
    v2 = v2 + 1;
}

fun4();     // nao precisa ter return pq esta sendo usado sozinho (mas pode ter)
fun5();     // mesma coisa: nao precisa ter


// fun6, fun7 e fun8 todos tem return pq estao sendo usados em if else
if (fun6() && fun7() || fun8()) {  
    v2 = v2 + 1;

    if (v2 == 10) {
        fun9();  // nao precisa ter return porque esta sendo usado sozinho
        v2 = fun10() * 2; // tem que ter return pq esta sendo usado em expressao
    }
}

Então basicamente é isso: se você pretende criar uma função e essa função vai ser usada em uma expressão ou num if else, então sua função DEVE OBRIGATORIAMENTE USAR RETURN!. Caso ela vá ser usada sozinha, então o return não é obrigatório.

Se você esquece de colocar um return numa função que deve ter return, teu programa não vai rodar direito. Você precisa entender que a função do return é retornar um valor para ser usado numa expressão, num cálculo. Expressões trabalham com valores. Agora pensa o seguinte: se tua função não retornar um valor, como é que a expressão vai ser calculada se tem valor faltando, justamente o valor que sua função deveria ter retornado e não retornou porque você esqueceu de usar return? E a resposta é: não vai rodar, simples assim.

Por isso, lembre: se vai usar a função numa expressão, ela deve ter return. É justamente por isso que as funções abaixo tem return.


function lf(msg) {
    return parseFloat(window.prompt(msg));
}

function li(msg) {
    return parseInt(window.prompt(msg));
}

function ls(msg) {
    return window.prompt(msg);
}

Porque elas são usadas em expressões:


var nome = ls('Qual o seu nome?');
var n1 = lf('Digite um float');
var i1 = li('Digite um inteiro');

if (li('Digite um outro inteiro') > 10) {
    console.log('numero eh maior que 10');
} else {
    console.log('numero n eh maior que 10');
}

Bom, um outro detalhe tão importante quanto saber quando ou não usar return, é saber o o que deve ser retornado. No caso das funções lf, li e ls o que eu quero que seja retornado é justamente o valor digitado pelo usuário. De tal forma que não faria sentido fazer algo assim, por exemplo:


function lf(msg) {
    parseFloat(window.prompt(msg));
    return 'nada a ver';
}

function li(msg) {
    var k = parseInt(window.prompt(msg));
    return 2 + 2;
}

function ls(msg) {
    window.prompt(msg);
}

Entende? Além de saber quando retornar, você deve saber o que retornar. Se você coloca pra retornar qualquer maluquice, teu programa vai rodar maluco também ou mesmo não funcionar, ok?

Então, o que sabemos que funções podem fazer por nós até agora:

  • Reutilizar código
  • Quebrar o código em partes que faça sentido para a gente
  • Deixar o código mais compacto e legível

Parâmetros

Talvez algo que você ainda esteja em dúvida sejam parâmetros: para que servem? quando usar? Bom, a primeira coisa para se entender os parâmetros é saber que não é obrigatório ter eles. Se uma função não tem parâmetros, a os parenteses da função ficam vazios. Ou seja, ao invés de ter algo como func1(a, b, c), fica simplesmente func1(). Parâmetros não são obrigatórios.

Ok, se parâmetros não são obrigatórios, então por que usar? Você usa os parâmetros quando quer que uma chamada à uma função seja diferente de outra chamada à mesma função. Para tentar entender melhor, vamos usar a seguinte função como exemplo:


function lf(msg) {
    return parseFloat(window.prompt(msg));
}

Como você já deve saber, eu criei essa função como uma abreviatura à parsefloat(window.prompt()). Essa função lf tem um parâmetro: msg, que é a mensagem que será mostrada pro usuário. Na prática a gente usa ela assim:


var n1 = lf('Digite um float');
var n2 = lf('Digite outro float');

window.prompt('Voce digitou os numeros ' + n1 + ' e ' + n2);

Ou seja, a cada chamada à lf(), quero que seja mostrado uma mensagem diferente. Essa diferença é conseguida justamente através do uso de parâmetros.

Um outro exemplo: suponha uma função para indicar se um aluno está reprovado por faltas ou não. Se ele tiver mais de 10 faltas ele está reprovado.


// funcao verificaReprovacao tem um parametro chamado faltas
function verificaReprovacao(faltas) {
    if (faltas > 10) {
        return 'esta reprovado'; // tem return pq vai ser usado em expressao
    } else {
        return 'nao esta reprovado';
    }
}

// uso da funcao li() ao inves de parseInt(window.prompt())
var f1 = li('Digite a quantidade de faltas do primeiro aluno');

var f3 = li('Digite a quantidade de faltas do terceiro aluno');
var f4 = li('Digite a quantidade de faltas do quarto aluno');
var f5 = li('Digite a quantidade de faltas do quinto aluno');

// verificaReprovacao sendo usado em expressao
window.prompt('O primeiro aluno ' + verificaReprovacao(f1));

window.prompt('O quarto aluno ' + verificaReprovacao(f4));
window.prompt('O quinto aluno ' + verificaReprovacao(f5));

Veja que foi usado parâmetro faltas para diferenciar uma chamada da outra. Cada chamada calcula o status de reprovacao de um aluno diferente. A função verificaReprovacao, por sua vez, retorna uma string indicando se o aluno está reprovado ou não. Essa string que é retornada é usada para construir as mensagens no fim do programa. Como funções são usadas em programas grandes?

Uma coisa legal de funções é que você não precisa saber como é o corpo dela para poder usar. Só relembrando, uma função é dividida da seguinte forma:


// OBS.: nome(parametros) tambem eh conhecido como assinatura da funcao, ou seja, o nome da funcao
//       e seus parametros

function nome(parametros) {
    corpo
}

Quando você usa window.prompt ou console.log, você está usando funções, ou seja, está chamando essas funções. Você provavelmente não sabe qual é o corpo das funções prompt e log. Mesmo assim é capaz de usar. Para usar uma função você precisa saber qual a assinatura dela (nome e os parametros), o que ela faz e se retorna algum valor ou não. Então, por exemplo, você sabe que prompt recebe um parâmetro e retorna uma string, enquanto que log também tem um parâmetro mas não retorna nada.

Programas grandes são dividos em várias e várias funções. Quando existe uma equipe envolvida, cada membro fica responsável por fazer parte das funções. Então, por exemplo, você pode encontrar algo assim no desenvolvimento de software para um celular. Ou seja, as funções permitem programadores trabalhando num mesmo projeto dividir um programa grande em partes menores e cada um cuidar de sua parte. Depois, basta combinar as funções para formar um programa maior.

Você já faz isso desde o início: alguém criou as funções prompt, parseInt, parseFloat, log, etc e você está usando elas para criar os seus próprios programas.

Por fim, para completar essa seção: quando você tem várias funções trabalhando em conjunto para atingir um objetivo, você tem uma biblioteca. Existem bibliotecas para auxiliar no desenvolvimento de jogos, de sites, de aplicativos para celular, etc. Uma biblioteca de jogo, por exemplo, vai te trazer uma série de funções já prontas para você poder utilizar para controlar os gráficos, o teclado, mouse, controles, som, etc.

Também existem os frameworks. Os frameworks também são um conjunto de funções com um um objetivo em comum. A diferença deles para bibliotecas é que nas bibliotecas você tem mais liberdade em decidir como usar e objetivo é menor. Já um framework tem regras mais rigidas de como deve ser usado, onde essas regras mais rigidas visam ter uma maior padronização de código e os objetivos são mais altos. Por exemplo, enquanto que você pode ter uma biblioteca de jogo para tratar só da parte gráfica, um framework de jogos pode vir com funções para a parte gráfica, som, rede, dados, etc etc etc. Ou seja, um framework geralmente é bem maior do que uma biblioteca. Mas no fim do dia tudo não passa de funções :)

Então esses jogos e programas que vocês costumam usar não são feitos do zero. Geralmente se usam bibliotecas e frameworks que trazem funções prontas para serem usadas, agilizando e auxiliando no desenvolvimento do software.

Um pouco mais de funções na prática

Uma coisa que você precisa ter bem em mente é a distinção entre a declaração de uma função e a chamada da função. Isso é declarar uma função:


function nome(parametros) {
    corpo
}

E isso é chamar a função:


nome(parametros);
var u = nome(parametros);

if (nome(parametros)) {
    ...
}

O que estou tentando dizer é que quando você declara uma função, ela não é automaticamente executada. Ela só irá executar SE E APENAS SE você chamar ela. Isso, na prática, significa que não vai importar a ordem que a gente declara as funções, mas sim que vai importar apenas a ordem em que a gente chama elas.

Considere as seguintes funções:


function calcularCarga(peso, altura) {
    return peso * 7 + altura * 1.3;
}

function ehMaiorDeIdade(idade) {
    if (idade >= 18) {
        return 'eh maior de idade';
    } else {
        return 'nao eh maior de idade';
    }
}

Se você escrevesse como está abaixo, teria exatamente o mesmo efeito no fim das contas, porque essas são as declarações das funções e não as chamadas às mesmas.


function ehMaiorDeIdade(idade) {
    if (idade >= 18) {
        return 'eh maior de idade';
    } else {
        return 'nao eh maior de idade';
    }
}

function calcularCarga(peso, altura) {
    return peso * 7 + altura * 1.3;
}

Por outro lado, tem uma enorme diferença entre fazer assim:


window.prompt('A carga eh de ' + calcularCarga(70, 1.8));
window.prompt('Fulano eh ' + ehMaiorDeIdade(20));

E fazer assim:


window.prompt('Fulano eh ' + ehMaiorDeIdade(20));
window.prompt('A carga eh de ' + calcularCarga(70, 1.8));

No primeiro é executado calcularCarga() e depois ehMaiorDeIdade() e no outro é o inverso: primeiro ehMaiorDeIdade() e depois calcularCarga(), o que vai gerar um resultado diferente entre as execuções. Novamente, isso só vai fazer sentido se estiver claro para você a distinção entre declarar uma função e executar a mesma.

Composição de funções

Um outro ponto bastante útil das funções é que uma função pode chamar outras funções que por sua vez podem chamar mais outras funções e assim por diante. Essa característica te permite fazer funções pequenas e depois ir combinando elas, ou seja, ir chamando umas as outras, para resolver problemas maiores e mais complexos.

De fato, quando a gente cria uma função tão simples como essa:


function lf(msg) {
    return parseFloat(window.prompt(msg));
}

Nós já temos uma composição de função: prompt() é chamada por parseFloat() que por sua vez será chamada por lf() e onde lf() será chamada em qualquer outra função onde seja necessário ler um float.

A propósito, quando se está lendo um texto e queremos nos referencia a uma função, é comum escrever a mesma com os () vazios, mesmo que ela possua parâmetros. No parágrafo anterior, eu me referi à prompt(), parseFloat() e lf() mesmo as três recebendo parâmetros (observe a assinatura da função lf(). Você ainda lembra o que é assinatura de uma função?)

Uma outra característica que é interessante você se atentar e procurar criar a cultura de usar, é compor funções nas chamadas. Por exemplo, considere o seguinte trecho de código:


function dobrar(x) {
    if (x > 100) {    
        return 2 * x;  // se x for maior que 100, a gente retorna o dobro de x
    } else {
        return x;      // caso contrário retorna o x inalterado
    } 
}

function quadruplicar_versao1(x) {
    return dobrar(dobrar(x));
}

function quadruplicar_versao2(x) {
    var p = dobrar(x);
    var z = dobrar(p);
    return z;
}

function quadruplicar_versao3(x) {
    var p = dobrar(x);
    return dobrar(p);
}

As três versões de quadruplicar produzem exatamente o mesmo resultado. Porém perceba que na primeira versão, você pode jogar o resultado de uma função diretamente para uma outra função, ou seja, composição na chamada. Você também pode usar variáveis auxiliares como na versão 2 da função quadruplicar ou um misto das duas estratégias, como na versão 3.

Humn... Mas porque não fazer sempre da primeira forma?. Porque pode acontecer algo como o trecho de código abaixo, que é feio de furar os olhos


return umaFuncaoDeNomeMuitoGrandeRuimDeLer(AiVemMaisOutraComNomeGrandeTambem(ComUmParametroGrandeTambem + 2, EseTiverMaisParametros(FicaMaisFeioAinda, AiMeusOlhos, AHH)));

Às vezes, você pode tentar compor funções que tenham nomes muito grandes e acabar ficando como uma coisa gigante e ruim de ler. Nesses casos, é preferível usar variáveis auxiliares e trocar uma linha gigante por várias linhas menores. Assim, por exemplo:


var p1 = ComUmParametroGrandeTambem + 2
var p2 = EseTiverMaisParametros(FicaMaisFeioAinda, AiMeusOlhos, AHH);
var p3 = AiVemMaisOutraComNomeGrandeTambem(p1, p2);
var p4 = umaFuncaoDeNomeMuitoGrandeRuimDeLer(p4);
return p4;

Como você pode perceber, não existe um jeito certo de criar funções. O que existem são formas de deixar mais organizado e isso vai da questão de boa estética e de bom gosto de cada um.
